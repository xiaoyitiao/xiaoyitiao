### 概念地图

#### Rust 是什么？
Rust 是一种**系统编程语言**，由 Mozilla 创造，主要目标：**让你写出既快又安全的代码**。

#### 传统语言的选择困境
```
传统语言的选择困境：
├── C/C++  →  快，但一不小心就会内存出错（崩溃、安全漏洞）
└── Java/Go → 安全，但有垃圾回收器拖慢速度

Rust 的答案：不用垃圾回收，也能内存安全
```

### 五个核心概念

#### 概念 1：所有权 (Ownership)

**一句话定义**：Rust 里每个值只有一个"主人"，主人没了，值就没了。

```rust
let s1 = String::from("hello");  // s1 是主人
let s2 = s1;                      // 所有权转移到 s2
// println!("{}", s1);  // ❌ 错误！s1 已经无效了
println!("{}", s2);              // ✅ OK
```

#### 概念 2：借用 (Borrowing)

**一句话定义**：你可以"借用"一个值，而不是拥有它。

```rust
let s1 = String::from("hello");

fn calculate_length(s: &String) -> usize {  // & 表示借用
    s.len()
}  // s 离开函数，但不影响 s1，因为只是借用

let len = calculate_length(&s1);
println!("{} 的长度是 {}", s1, len);  // ✅ s1 仍然有效
```

#### 概念 3：生命周期 (Lifetimes)

**一句话定义**：确保引用不会指向已经不存在的东西。

```rust
// 这个函数返回的是哪个 x 的引用？
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

#### 概念 4：trait（特征）

**一句话定义**：定义可以被多个类型共享的行为，类似于其他语言的"接口"。

```rust
trait Speak {
    fn speak(&self);
}

struct Dog;
struct Cat;

impl Speak for Dog {
    fn speak(&self) { println!("汪！"); }
}

impl Speak for Cat {
    fn speak(&self) { println!("喵！"); }
}
```

#### 概念 5：Rust 的"三合一"哲学

```
Rust = 性能 + 安全 + 并发
       (Performance + Safety + Concurrency)
```

### Level 1: 记忆

**问题 1**：Rust 的 Ownership 规则是什么？值只能有多少个"主人"？
> 回答：每个值有一个所有者、同一时间只有一个所有者、所有者离开作用域值被释放；值只能有1个主人

**问题 2**：借用和拥有有什么区别？请举例说明。
> 回答：拥有是拥有值的所有权，决定值的生命周期；借用是临时使用权，不拥有所有权

**问题 3**：Rust 为什么不需要垃圾回收器也能自动清理内存？
> 回答：通过所有权、借用和生命周期的编译期规则，在编译时确定值的生命周期并自动插入清理代码，无需运行时垃圾回收

✅ Level 1: 通过

---

### Level 2: 理解

**问题 1**：如果 Rust 的 Ownership 是一个"收快递"的规则，你会怎么描述它？
> 回答：一个快递只有一个收件人，不能同时好多人取同一个件

**问题 2**：为什么 Rust 要设计"借用"这个概念？直接拥有不好吗？
> 回答：我的书只能我决定借给谁看，什么时候借让朋友爱护我说了算

**问题 3**：Lifetime `'a` 中的 `'a` 是什么意思？
> 回答：清理空间，不能物件一直堆着占空间

部分理解，需要深入：
- 借用 = 暂时给别人用，**用完还给你**，你还是主人
- Lifetime 的目的不是"清理"，而是**防止你用一个已经不存在的东西**

### Level 3: 应用

**题目**：修复代码，使其能正确打印两个数组的长度。

```rust
fn main() {
    let arr1 = &vec![1, 2, 3, 4, 5];
    let arr2 = &vec![6, 7, 8, 9, 10];

    println!("arr1 的长度是: {}", arr1.len());
    println!("arr2 的长度是: {}", arr2.len());
}
```

> 回答：添加 `&` 符号进行借用

✅ Level 3: 通过

---

### Level 4: 分析

**问题**：为什么 `data.push(6)` 会被禁止？

```rust
let mut data = vec![1, 2, 3, 4, 5];
let first = &data;        // 借用 data
data.push(6);              // ❌ 编译错误！为什么？
println!("{}", first);
```

> 回答：可变引用与不可变引用的冲突

深入分析：
- `Vec` 在 push 时可能会重新分配内存
- 导致原来的引用失效，访问已释放的内存 = 悬垂指针
- Rust 编译时就拦住，根本不给你机会

```
表层：Rust borrowing rules "同时有不可变引用和可变引用？不行！"
深层：防止悬垂指针 (dangling pointer) "如果允许，first 就变成了野指针"
```

✅ Level 4: 通过
