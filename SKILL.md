# 快速掌握任何主题 (Rapid Mastery)

基于认知科学和学习科学的主题掌握方法论。与 Claude Code 协同使用，通过"主动建构 + AI 增强"实现高效学习。

---

## 核心理念

> **学习不是输入，而是建构**  
> **理解不是记住，而是能教**  
> **掌握不是完成，而是能用它解决真实问题**

---

## 五阶段学习法

### Phase 1: 诊断定位 (5-10 min)

**目标**: 精准定位起点和终点

1. **评估现有知识** - 用 3-5 个诊断问题确定基础
2. **明确学习目标** - 用输出定义："学完能做什么"
3. **设定时间边界** - 决定学习深度和范围

**参考**: [phase-guides/phase1-diagnose.md](phase-guides/phase1-diagnose.md)

---

### Phase 2: 搭建脚手架 (10-15 min)

**目标**: 建立整体认知框架

1. **生成知识地图** - 核心概念 + 概念关系 + 常见误解
2. **识别 20% 核心** - 帕累托原则，聚焦高价值内容
3. **规划学习路径** - 从已知到未知的最佳路线

**参考**: [phase-guides/phase2-scaffold.md](phase-guides/phase2-scaffold.md)

---

### Phase 3: 主动探索 (主要阶段)

**目标**: 通过"预测-实践-反馈"循环深化理解

**核心循环**:
```
预测 → 实践 → 对比 → 修正 → 迁移
```

1. **先猜后学** - 对每新概念，先让用户预测/尝试
2. **多模态学习** - 读、写、画、跑代码、做实验
3. **AI 苏格拉底** - Claude 提问引导，而非直接给答案
4. **即时应用** - 每学一点，立即在项目中使用

**参考**: [phase-guides/phase3-explore.md](phase-guides/phase3-explore.md)

---

### Phase 4: 输出与检验 (20-30 min)

**目标**: 用输出倒逼输入，暴露理解盲区

1. **最小可行项目** - 解决一个真实小问题
2. **教给 Claude** - 用户当老师，Claude 当学生提问
3. **创建知识卡片** - 为间隔复习准备素材
4. **自我评估** - 对照最初目标，检查掌握程度

**参考**: [phase-guides/phase4-output.md](phase-guides/phase4-output.md)  
**模板**: [templates/concept-card-template.md](templates/concept-card-template.md)

---

### Phase 5: 间隔巩固 (长期)

**目标**: 对抗遗忘，内化为长期记忆

1. **24 小时回顾** - 不看资料，向 Claude 复述
2. **一周后应用** - 用新技能解决新问题
3. **一月后重构** - 重新绘制知识地图，看到成长

**工具**: [scripts/generate_review_schedule.py](scripts/generate_review_schedule.py)

## 使用流程

```
用户: "我想快速掌握 [主题]"

1. 执行 Phase 1: 诊断定位
   - 问 3-5 个诊断问题
   - 明确学习目标和时间

2. 执行 Phase 2: 搭建脚手架
   - 生成知识地图
   - 确定 20% 核心内容

3. 执行 Phase 3: 主动探索
   - 对每个核心概念循环：预测→实践→反馈
   - 边学边在项目中应用

4. 执行 Phase 4: 输出与检验
   - 完成最小可行项目
   - 教给 Claude
   - 创建知识卡片

5. 执行 Phase 5: 间隔巩固
   - 生成复习计划
   - 设置回顾提醒
```

## 学习理论根基

详见: [references/learning-theories.md](references/learning-theories.md)

| 理论 | 核心原则 | 在本 Skill 中的应用 |
|------|----------|---------------------|
| **建构主义** | 知识是主动建构的 | 先猜后学、AI 作为协作者 |
| **做中学** | 实践是理解的唯一路径 | 边学边做项目 |
| **间隔重复** | 对抗遗忘曲线 | 24h/1周/1月 回顾机制 |
| **主动回忆** | 测试效应 > 重复阅读 | 教给 Claude、自我测试 |
| **费曼技巧** | 教是最好的学 | 向 Claude 解释概念 |
| **生成式学习** | 主动生成促进理解 | 让用户用自己的话复述 |
| **元认知** | 监控自己的理解 | 反思"我怎么想到的" |
| **支架式教学** | 动态调整支持程度 | 根据水平调整提示深度 |
| **精细加工** | 建立新旧知识连接 | 类比迁移、多角度关联 |
| **分布式认知** | 外部工具扩展认知 | 用 Claude 存储和计算 |

## 适用场景

✅ **技术类**
- 编程语言 (Rust, Go, Python...)
- 框架/库 (React, PyTorch, Spring...)
- 工具使用 (Docker, Kubernetes, Git...)
- 系统设计概念

✅ **学科类**
- 数学概念 (微积分、线性代数...)
- 科学原理 (物理、化学...)
- 统计/机器学习概念

✅ **技能类**
- 设计工具 (Figma, Blender...)
- 写作/演讲技巧
- 软技能 (沟通、时间管理...)

❌ **不适用场景**
- 需要深厚理论根基的学术研究
- 纯记忆性内容 (建议用专门的记忆工具)
- 考试/面试突击 (可增加更多专项练习)

## 与 Claude Code 协同

| 阶段 | Claude 做什么 | 用户做什么 |
|------|--------------|-----------|
| 诊断 | 设计诊断问题、分析回答 | 回答问题、确认目标 |
| 脚手架 | 生成知识地图、标记核心 | 审查、选择重点 |
| 探索 | 苏格拉底式提问、生成示例 | 预测、实践、反思 |
| 输出 | 扮演学生提问、评估项目 | 解释、教学、创建卡片 |
| 巩固 | 生成复习计划、评估复述 | 主动回忆、自我测试 |

## 关键提示词模板

### 诊断阶段
```
"为了帮你高效学习 [主题]，我需要了解：
1. 你已经了解哪些相关知识？
2. 你想用 [主题] 解决什么问题？
3. 你有多少时间投入？"
```

### 探索阶段
```
"在学习 [概念] 之前，先猜猜看：你觉得它是做什么的？
不用怕错，猜测本身就是学习过程。"
```

### 输出阶段
```
"现在请你向我解释 [概念]，我会扮演一个初学者，
如果我有不懂的地方会随时提问。"
```

## 核心心态

> "看视频/文档是输入，做项目是输出。只看不练等于没学。"
>
> "理解一段代码的最好方式是改坏它再修好。"
>
> "教是最好的学——向 Claude 解释，直到它没有问题。"
>
> "AI 是脚手架，不是拐杖。逐步减少对 AI 的依赖。"
