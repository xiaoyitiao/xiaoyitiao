一 选题与真实需求回顾
随着我国人口老龄化程度的加深，老年人用药安全问题日益突出。据调研数据显示，65岁以上老年人平均漏服率达30%-50%，多药同服时药物相互作用风险显著增加，漏服或误服药物可能导致病情加重或引发并发症。子女在外工作，无法实时监督父母用药情况，这成为众多家庭的痛点。
核心痛点分析
问题类别 具体表现 影响程度
操作不便 老年人视力下降，传统手机应用字体小、操作复杂 高
记忆困难 记忆力衰退，容易忘记服药时间和剂量 高
安全隐患 多种药物同时服用时，缺乏药物相互作用提醒 极高
亲情缺失 子女无法远程了解父母的用药情况 中
解决方案定位：
智药伴旨在打造一款专为老年人设计的用药提醒应用，核心价值主张——"让用药更简单，让关爱更及时"。通过语音交互、大字显示、智能提醒等特性，降低老年人使用门槛；通过家属端功能，让子女远程掌握父母用药状态，构建完整的关爱闭环。

核心技术实现
前端架构：采用单页应用架构，基于原生JavaScript开发，无需额外框架，轻量高效。核心文件包括index.html（应用入口）、style.css（核心样式）、app.js（主应用逻辑），以及各功能模块的独立JS文件。
数据存储方案：采用本地优先的存储策略。IndexedDB用于存储药品数据、用药记录和用户设置，容量大且支持事务操作；localStorage用于存储简单配置项；敏感数据采用AES加密存储，密钥基于设备特征派生，确保数据安全。
语音交互实现：使用Web Speech API实现语音识别，支持中文语音转文字；使用SpeechSynthesis API实现语音合成，语速调慢至0.85倍，适合老年人听觉习惯；支持语音导航，说出指令即可自动跳转页面。
响应式设计：以手机屏幕尺寸为基准设计，使用Flexbox和Grid实现自适应布局；按钮尺寸≥48px，便于手指操作；支持三档字体大小切换，默认大字模式。

五 反思与改进
开发中遇到的问题与解决方式
  • 移动端兼容性问题
    iOS Safari对Web Speech API支持有限，语音识别功能无法正常使用
    解决：检测浏览器类型，iOS端降级为手动输入模式，提供友好提示
  • 数据安全问题
    本地存储可能被非法访问，敏感数据存在泄露风险
    解决：采用AES加密存储敏感数据，密钥基于设备特征派生，不离开设备
  • GitHub Pages部署问题
    资源路径配置错误，页面无法正确加载CSS和JS文件
    解决：修改Vite配置，使用相对路径base: ''，确保资源正确加载
  • 样式显示问题
    缺失核心CSS文件，页面显示为纯文本，没有任何样式
    解决：重新编写完整的style.css，包含所有组件样式，确保页面正常显示
  • 弹窗点击失效
    多个模态框重叠，点击事件被上层元素拦截，隐私协议同意按钮无法点击
    解决：修复CSS，确保只有需要的弹窗显示，其他默认隐藏，添加.show类控制显示状态
